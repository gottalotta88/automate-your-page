Stage 1: Make Your Webpage

Lesson 1: Basics of web and HTML

Big ideas

HTML: Hypertext markup language

Web invented in the 1990s; all pages link together

Browser makes requests to the internet via HTTP to servers, which respond with files that the browser displays

Components of HTML

Text content: what you see

Markup: what it looks like

References to other documents: images and videos

Test out code at link

Opening tags and closing tags; both tags plus contents within is called an element

Notes on elements

B tag is for bold

em tag is for italics

If you don't close tag; everything after is bolded/italicized

a is for anchor

  • Used for making links
  • Followed by an attribute name, value, and then contents
img is for image
  • Src and alt are attributes; alt is what to display if broken request
  • Img doesn't have any content, so no closing tag; this is called a void tag, a tag that has no content
Lines
  • br is a line break to make text go onto next line; also a void tag
  • p is for paragraph; need closing tag after content
  • Br just puts to next line whereas p tag makes an invisible box; br is an inline element while p is a block element; latter creates blocks on the screen
  • Spaces and returns will not be registered
Span and div
  • Span can be used to give CSS styles to contents; inline element
  • Div does the same but as a block element
Thus basic HTML vocab consists of tags, elements, and attributes

Allows style and appearance formatting; for example can make all h1 elements of the class "x" look the same way

Repetition also makes a page run faster

Lesson 3: Adding CSS Style and HTML Structure

Basic terms

CSS: Cascading style sheets

Inheritance: descended elements nested within an HTML element will inherit CSS properties of the larger element, unless the sub-element has its own specified CSS style

Basic structure

For all h1: h1 {color:red;}

For specific CSS classes: .classname {color:red;}

h1 is a selector; rule can be applied to all h1; can also apply rule to all div

color:red is a declaration

color is a property; red is the value of that property

Mozilla has good exhaustive reference for CSS properties; e.g. font-weight specifies boldness

Outline CSS property can be used to better see block structure of your page

Clarifications
Span vs. Div
  • Div is block element; good for grouping and organization
  • Span class can be used as inline element; example in class code of using it for span class=italic and then defining italic in CSS
Id vs class
  • Id uniquely identifies an element and don't repeat, as opposed to class which can be repeated
  • Useful if want to use anchor element to link to a particular spot; id="welcome"; a href="#welcome"
  • Class better for grouping similarly
  • Can use both id and class in a div
Box model

All HTML elements are boxes

Components
  • Content - text and images
  • Padding - transparent area
  • Border - border around padding and content
  • Margin; transparent; basically whitespace between boxes

Width of content as defined won't be total width; total width of box involves adding content, padding, and border

Not intuitive to have defined width not reflect what is actually rendered; thus box-sizing property with border-box value will make defined width the width of the combo of border, padding, and content which is more intuitive; margin not included in size

Can set width as percentage of window so that it adjusts to window width

Max-width attribute such that will adjust but not above a max width; overrides width property

Lesson 2: Variables and strings

Variables

Assignment statement assigns value to a variable

Can compute by connecting assigned variables by operators

The equals sign in python means assignment, not equals

Strings

Can use double or single quotes

Convention to not capitalize names of variables in python

Concatenation refers to adding two strings together

Can also use multiplication operator to repeat a string

Indexing strings

Indexing refers to retrieving a value of a particular character in a string

s[0] gives first letter of string named s; ditto for subsequent numbers; if s[3] and string only 3 characters, will return error; s[0] can cause error for empty string; subsequence operator does not return error for this issue

s[-1] gives last letter

Can also do "example"[0] which will result in e, where string is defined in the expression

Subsequences can be extracted with expression like s[0:2] which will represent first 2 letters; starts at the first number, ends before the second number

Can also do s[:2] which will give first 2 characters; key is doesn"t include the number you border with; can do s[2:] which goes to end

Just like with lists, can use in/not to index and return boolean value for whether substring is within a string; print "hello" in y returns True is string y contains hello

Finding strings in strings

s.find(string); passing a string to this function, will return position of that string; if not found, will return -1

Can pass in an extra parameter; s.string("s", 1); this will search starting at the number you specified

Additional info

Multiline strings: can make these with triple quotes

Abstract thinking involves producing solutions that apply to general situations more so than specific ones

Lesson 6: Structured Data: Lists and For Loops

Lists

Lists contain multiple ordered elements which can include numbers, strings, or other lists

When assigning list, enclose in brackets; tuples are enclosed in parentheses

Can have nested lists where every element in a list is another list; this is great for storing items when have items that have attributes like name and birthdate or country and capital

Mutation refers to the ability to change individual elements within lists by assigning new values to them

Strings are not mutable whereas lists are; numbers are also not mutable since they are not objects

If assign a list p to a different variable q, now both variables refer to the same list object; if you mutate p, q will experience that mutation since both pointing to the same object

Having two different ways to refer to the same object is called aliasing

List operations for a list p

Append: .append() will add string to the list; can also append with a new list; this appended element will then refer to the list and respond to any changes in this list object to which it refers

Addition: + ; key distinction between append and add is that former adds new item as a list item

Length: len() gives length of the list; this also works for strings to get number of characters

Index: .index() will return location; if not present, get error; you wrote a longer way to do this in the exercise;

In/Not in: in ; output boolean answer to whether true or false; can also do not in to give the opposite boolean value; when using in if statement, don"t need any additional operators since returns boolean; "if x in p"

For loops

Can theoretically use a while loop with a i < len(p) criteria to walk down a list but for is available to do this with less code

"For x in y" has the function walk along the list y and then you can do whatever you want to each element x; if there are nested lists, can manipulate them within function using list operations
